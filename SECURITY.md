# Security Policy

## Reporting

Do not open public GitHub issues for security-sensitive bugs.

Instead, report them privately to the maintainer with:

- a concise description of the issue
- affected files or commands
- reproduction steps
- impact assessment

## Scope

Security reports are especially useful for:

- native binding memory safety issues
- unsafe file handling in scripts
- training or rollout paths that expose unintended code execution surfaces
- dependency or packaging issues that materially affect users

## Response

Reports will be triaged and fixed based on severity, exploitability, and maintenance bandwidth. Coordinated disclosure is preferred.
